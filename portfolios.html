DAO Hardhat, Hardhat-deploy, Solidity, TypeScript, JavaScript, Slither. - https://github.com/SashaFlores/DAO
  • This project is an on-chain DAO that utilizes a Governor Controller to manage voting for proposals with:
    1. VOTING_DELAY of 1 block, meaning that a proposal can be voted on one block after it has been proposed.
    2. VOTING_PERIOD is set to 5 blocks, indicating that the voting period for each proposal is 5 blocks long.
    3. QUORUM_PERCENTAGE is set to 4%, which means that at least 4% of the total token holders need to vote in order for a proposal to be considered valid.
  • The project also includes a TimeLock mechanism, which sets a minimum delay of 3600 seconds (1 hour) for executing any proposed changes. The EXECUTORS and PROPOSERS arrays are used to specify the addresses of individuals or entities that can execute or propose changes, respectively.
  • Overall, the project seeks to create an on-chain DAO that allows for transparent and democratic decision-making through voting on proposals. The Governor Controller, TimeLock mechanism, and script proposals ensure that proposals are rigorously reviewed and executed in a secure and transparent manner.
